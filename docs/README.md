# Project Documents

- `MiniProject_HPC_Flowtable.docx`: original source document kept at the repo root.
- `docx_content.txt`: extracted text from the `.docx` for quick terminal review.
- `project_architecture.md`: implementation-oriented architecture notes for the project.
- `spi_rule_engine_design.md`: SPI rule engine design and hot-reload notes.
- `source_code_defense_guide.md`: source-code-level explanation of runtime behavior, data structures, optimizations, and current limitations.
- `cache_and_datapath_explained.md`: packet-path, memory-access, and cache-behavior explanation tied directly to the current source.
- `app_defense_manual.md`: final defense-oriented document covering architecture, structs, functions, datapath, concurrency, flow table, RCU, SPI, aging, monitoring, and optimizations.

The goal of this directory is to keep design and reference material out of the repo root while leaving runtime assets, scripts, and source code easier to scan.
